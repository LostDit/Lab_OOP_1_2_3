## [Лабораторная работа 1 — Структуры и работа с файлами](https://github.com/LostDit/Lab_OOP_1_2_3/tree/main/Lab_1) ✅

**Цель:** работа со структурами, перегрузка операторов ввода/вывода, динамические массивы, функции поиска и фильтрации, бинарные и текстовые файлы.

### Реализовано
- Структуры `Fullname` и `Employee` с перегруженными `<<` и `>>`.
- Инициализация массива сотрудников с клавиатуры или из текстового файла.
- Вывод списка сотрудников на экран, в текстовый и бинарный файл.
- Функции поиска:
  - `find_element` – первое вхождение по условию.
  - `find_all_elements` – выборка всех подходящих записей.
  - `needful_element` – поиск экстремального элемента по компаратору.
- Пользовательское меню для выбора действий.
- Поддержка вектора вместо массива (для оценки «отлично»).

### Задания по вариантам
| № | Вариант 1 | Вариант 2 |
|---|-----------|-----------|
| 1 | Проверить, работают ли женщины. | Проверить, работают ли мужчины. |
| 2 | Найти самую молодую сотрудницу. | Найти самого пожилого сотрудника. |
| 3 | Определить наличие женщин-пенсионерок (пенсия с 60 лет). | Определить наличие мужчин-пенсионеров (пенсия с 65 лет). |

## [Лабораторная работа 2 — Классы `Radio`, `Field`, `Statistic` ](https://github.com/LostDit/Lab_OOP_1_2_3/tree/main/Lab_2)✅

**Цель:** инкапсуляция, конструкторы, геттеры/сеттеры, методы управления состоянием.

### Класс `Radio`
- Поля: громкость (`0–10`), радиостанция (`1–10`).
- Методы:
  - `incVolume()` / `decVolume()`
  - `nextStation()` / `previousStation()` (циклическое переключение)
- Защита от выхода за допустимые границы.
- Управление с клавиатуры: **W/S** — громкость, **A/D** — станции (через `_getch()`).

### Класс `Field` (поле для «Пятнашек»)
- Размер задаётся параметром (по умолчанию 4×4).
- Данные хранятся в одномерном массиве + индекс пустой клетки.
- Методы сдвига: `shiftUp()`, `shiftDown()`, `shiftLeft()`, `shiftRight()`.

### Класс `Statistic` (анализ продаж)
- Данные хранятся в `std::vector<int>`.
- Расчёт суммы, среднего, количества месяцев выше/ниже среднего.
- Поиск месяцев с максимальными и минимальными продажами.
- Конструктор из вектора, деструктор.

### Класс `Fifteen`

- Запрет копирования и присваивания (приватные конструктор копирования и оператор `=`).

### Расширение `Statistic`
- Новые конструкторы:
  - `Statistic(const int*, size_t)`
  - `Statistic(const std::string&)`

## [Лабораторная работа 4 — Перегрузка операторов](https://github.com/LostDit/Lab_OOP_1_2_3/tree/main/Lab_4) ✅

### `Radio`
- Префиксные и постфиксные `++` / `--` для переключения станций.

### `Field`
- Операторы `==` и `!=` (используются при проверке окончания игры).

### `Statistic`
- Перегрузка `operator[]` (константный и неконстантный) с выбрасыванием `std::exception` при выходе за границы.
- Метод `getSlice(start, finish)` — получение среза статистики.
- Оператор `+=` для объединения статистик.
- Для оценки «отлично»:
  - `deleteSlice(start, finish)`
  - Операторы `==` / `!=` (сравнение по составу)
  - Дружественные `<<` и `>>`
  - Перегрузка `[]` для получения среза: `statistic[start][finish]`

## [Лабораторная работа 5 — Библиотека `Fraction` и модульное тестирование](https://github.com/LostDit/Lab_OOP_1_2_3/tree/main/Lab_5/FractionSolution)✅

### Структура
- **FractionLib** – статическая библиотека с классом `Fraction`.
- **FractionUnitTest** – проект модульных тестов (Microsoft Unit Test Framework).

### Класс `Fraction`
- Несократимая рациональная дробь (числитель — целое, знаменатель — натуральное).
- Конструкторы:
  - По умолчанию (`0/1`)
  - С одним или двумя целыми параметрами (автосокращение, проверка знаменателя ≠ 0)
  - Из строки вида `"a/b"`
- Перегрузка операторов: `+=`, `-=`, `*=`, `/=`, унарный минус, бинарные `+`, `-`, `*`, `/`.
- Операторы сравнения: `==`, `!=`, `<`, `>`, `<=`, `>=`.
- Операторы ввода/вывода (`<<`, `>>`).
- Методы:
  - `pow(unsigned)` – возведение в натуральную степень.
  - `inverse()` – обратная дробь.
- Умножение на целое число слева и справа.
- Неявное приведение к `double`, явное к `std::string`.
- Оптимизация вычислений для предотвращения переполнения (`отлично`).

### Класс `Indexer` (доп. задание «отлично»)
- Обёртка над массивом `double[]` для доступа к подмассиву.
- Контроль границ, исключения при некорректных параметрах.
- Перегрузка `operator[]`.

### Тестирование
Полное покрытие модульными тестами всех конструкторов, методов и операторов.

## [Лабораторная работа 6 — Линейные уравнения и системы](https://github.com/LostDit/Lab_OOP_1_2_3/tree/main/Lab_5/FractionSolution)✅

### Класс `LinearEquation` («хорошо»)
- Линейное уравнение от *n* переменных с вещественными коэффициентами.
- Конструкторы из строки, массива/вектора.
- Метод случайного заполнения коэффициентов.
- Перегрузка:
  - Унарный минус, `+=`, `-=`, `+`, `-`, умножение на число.
  - `==`, `!=`, `operator[]`.
  - Неявное преобразование к `std::vector<double>`.
  - `<<` и `>>`.
- Покрытие модульными тестами.

### Класс `SystemOfLinearEquation` («отлично»)
- Система из *m* уравнений с *n* переменными.
- Конструктор с указанием количества переменных.
- `operator[]` для доступа к уравнению по индексу.
- Метод приведения к ступенчатому виду и решение методом Гаусса.
- Обработка случаев:
  - Единственное решение.
  - Бесконечное множество решений.
  - Несовместность.
- Модульные тесты.

### Модификация `Statistic` («отлично»)
- Замена `std::vector<int>` на динамический массив `int*`.
- Адаптация всех методов, перегрузок и тестов.
